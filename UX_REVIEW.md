## Инструкция: “команда агентов” для проекта

Этот документ — единый источник правды для пересмотра UX продукта и постановки задач.  
Формат обсуждения каждой фичи одинаковый (см. “Шаблон фичи” ниже).

### Роли (агенты), которые участвуют в ревью

- **UI/UX дизайнер**: IA/навигация, визуальная иерархия, состояния, текстовые подсказки, “progressive disclosure”.
- **UX‑психолог (поведенческий)**: когнитивная нагрузка, мотивация/триггеры, “первые 30 секунд”, снижение тревожности, предотвращение ошибок.
- **SEO / контент‑стратег**: страницы для индексации (публичные), семантика контента, сниппеты, share‑preview, микро‑копирайт.
- **A11y‑специалист**: навигация с клавиатуры, aria‑лейблы, фокус‑стейты, контраст, touch targets.
- **Frontend‑архитектор**: роутинг, deeplink/redirect, состояния/данные, перфоманс, DX, тестируемость.
- **QA (тест‑инженер)**: e2e сценарии, “флейки”, негативные кейсы, регрессии, тест‑план.

### Правила совместной работы по фичам

- **Одна фича = один автономный блок** (можно разворачивать/прятать advanced), без “простыней”.
- **Сначала цель, потом действие**: вверху фичи всегда короткая цель и следующий шаг.
- **Прямые ссылки не ломаем**: при перестройке IA оставляем алиасы/редиректы и фиксируем их в “Технических деталях”.
- **Язык единый**: RU/EN строки только через i18n; не смешивать.
- **No secrets in UI**: токены/ключи не просим в обычном UI (только Advanced или автопроцессы).
- **Состояния обязательны**: loading/empty/error/success/disabled — в явном виде.
- **Метрика успеха**: для каждой фичи фиксируем “что значит лучше” (скорость, конверсия, понимание, удержание).

## Инвентаризация текущих фич (что есть в продукте сейчас)

Ниже — список фич, которые мы обязаны пройти. Для каждой дальше будет отдельный блок по шаблону.

### Навигация / IA (глобально)

- **Shell / Header**: шапка, тема, язык TMDB, выбор оформления (“лук”), индикатор сервера.
- **Account‑центр** (`/account/*`): вкладки “Сегодня/Дневник/Списки/Входящие/Настройки”.
- **Legacy redirects**: `/decide`, `/diary`, `/collections`, `/watchlist`, `/inbox`, `/notifications` (как алиасы/редиректы).

### Discover / Главная (`/`)

- **Поиск фильмов** (основной сценарий)
- **Секции**: “Сейчас в кино” (`/now-playing`), “Рекомендации” (`/recommendations`), “Random” (`/random`)
- **Карточки фильмов** (быстрые действия, hover‑actions)

### Movie details (`/movie/:id`)

- **Детали фильма**: постер/описание/кредиты/ключевые слова/локаль TMDB.
- **Действия**: статусы/избранное/дневник/подписки на релиз (если подключено).

### Today / Сегодня (внутри аккаунта) (`/account/today`)

- **Decision mode**: ограничения → shortlist → выбор победителя.

### Diary / Дневник (внутри аккаунта) (`/account/diary`)

- **Лента просмотров**, добавление записи, статистика.

### Lists / Списки (внутри аккаунта) (`/account/lists`, `/account/lists/statuses`)

- **Хаб**: табы “Статусы (watchlist)” и “Коллекции”.
- **Watchlist/Statuses**: want/watching/watched/dropped/hidden + bulk actions.
- **Collections**: ручные подборки.

### Inbox / Входящие (внутри аккаунта) (`/account/inbox`)

- **Feed** уведомлений (локальные/серверные), загрузка серверного фида.
- **Rules / subscriptions** (если есть в текущем UI).

### Account / Настройки (внутри аккаунта) (`/account/settings`)

- **Overview**: быстрые входы (share cards, me‑hub, кто залогинен).
- **Public profile**: slug, видимость, секции, контент “О себе/Заметки/Планы”, предпросмотр публичной страницы (`/u/:slug`).
- **Looks / Оформление**: выбор, unlock, создание/редактирование/удаление.
- **Server / Connections**: подключение к серверу, Web Push, dev email.
- **Data & Privacy**: экспорт, мастер приватности, advanced server tools (features cache, embeddings jobs).
- **Subscriptions**: управление подписками релизов.
- **Streaming prefs**: регион/провайдеры + каталог.
- **Favorites**: список избранного.

### Public profile (`/u/:slug`)

- **Публичная страница**: секции (favorites/diary/watchlist) + текстовый контент.
- **Share‑friendly**: цели “поделиться ссылкой” и “внешний просмотр”.

### Share cards (`/share`)

- **Генерация карточек** для шаринга (визуальные пресеты/лейауты).

### Me hub (`/me`)

- **Личный хаб** (набор “моих” страниц/инструментов).

### Import (`/import`)

- **Импорт данных** (источники, подсказки, ошибки).

### Favorites (`/favorites`)

- **Отдельная страница избранного** (если нужна/оставляем — решим в ревью IA).

## Ревью фич (заполняем по шаблону)

### 1) Навигация / IA (глобально)

1. **Существующая фича**
   - Глобальная шапка + основные точки входа, вложенная навигация внутри `/account/*`, редиректы со старых URL.

2. **Как сейчас (мнения агентов)**
   - **UI/UX**
     - В шапке осталось **2 пункта**: `Главная` (`/`) и `Аккаунт` (`/account`), остальное живёт во вкладках аккаунта.
     - `/account` — **хаб** с вкладками: `Сегодня / Дневник / Списки / Входящие / (Настройки)`; вход на `/account` редиректит на `/account/today`.
     - Плюс “Server pill” в шапке ведёт на `/account/settings#account-data`.
     - Сильная сторона: минималистично и без шума. Главная проблема: **перегруз термина “Аккаунт”** (внутри него не только настройки/профиль, но и главные рабочие разделы).
   - **Психолог**
     - Двухэтажная навигация (Header → Account → Tabs) добавляет “налог на ориентирование” в первые 30 секунд.
     - Слово “Аккаунт” вызывает ожидание “настройки/логин”, а не “основная работа” → возникает тревога “я не туда попал”.
     - “Сегодня” без глагола может считываться как “сводка дня”, а не “выбор фильма”.
   - **SEO**
     - Публичные/индексируемые маршруты должны быть отделены от приватных:
       - Индексировать: `/movie/:id*`, `/u/:slug` (и будущие публичные списки пользователя).
       - Не индексировать: `/account/*`, `/inbox*`, `/share` (инструмент), персональные/рандомные фиды.
     - Для SEO лучше иметь канонические URL и мета‑теги на публичных страницах; редиректы помогают избежать дублей.
   - **A11y**
     - Для “вкладок‑навигации” по роутам всё ок как `<nav>` + ссылки, но нужно:
       - активный пункт объявлять (`aria-current="page"`),
       - явные `:focus-visible`,
       - минимальные touch targets ≥44px.
     - В приложении есть места с “полу‑tab” семантикой (tablist/tab/tabpanel без полного паттерна) и bottom-sheet без полноценного focus-trap — это P0 для доступности (см. отдельные фичи, но влияет на глобальный UX).
   - **Frontend**
     - Редиректы на `/account/*` реализованы через `redirectTo` (могут **не сохранять** query/fragment).
     - Есть реальная тех. проблема: `/account/settings#account-data` **не работает стабильно**, потому что якорь существует только когда внутренняя вкладка настроек = `data`.
     - `lists` хаб определяет активную вкладку через `url.includes(...)` — хрупко.
   - **QA**
     - Нужны регресс‑проверки редиректов (в т.ч. цепочка `/notifications` → `/account/inbox`) и smoke‑тест табов `/account/*`.
     - Флейки: ожидание редиректа/сетевой стабильности; тестам надо ждать финальный URL.

3. **Существующие недостатки**
   - **Семантическая ошибка**: “Аккаунт” = “всё моё” (рабочие разделы) и одновременно “настройки” → когнитивный диссонанс.
   - **Дублирование слова/смысла**: вкладка `/account/settings` сейчас подписана как “Аккаунт” (получается “Аккаунт → Аккаунт”).
   - **Слабая “первичная подсказка”**: не очевидно, где начать (поиск vs выбор на сегодня vs логирование).
   - **Deep‑link баг**: `Server pill` ведёт на `#account-data`, но вкладка “Data” по умолчанию не открыта → якорь часто не срабатывает.
   - **Хрупкость табов Lists**: строковые `includes('/statuses')` и `includes('/account/lists')`.

4. **Цель данной фичи**
   - Сделать навигацию **целе‑ориентированной**: пользователь понимает “куда жать” за 10 секунд.
   - Уменьшить когнитивную нагрузку, убрать ложные ожидания от слова “Аккаунт”.
   - Сохранить стабильные ссылки и предсказуемый back/forward.

5. **Как исправить недостатки (план изменений)**
   - **Переименовать смысловые элементы**
     - Вкладку `/account/settings` назвать **“Настройки”** (а не “Аккаунт”).
     - Рассмотреть переименование верхнего “Аккаунт” в **“Моё” / “Библиотека”** (если хотим, чтобы там жили рабочие разделы).
     - Уточнить “Сегодня” → **глагол**: “Выбрать на вечер / Решить / Tonight pick” (минимум в подписи/подзаголовке).
   - **Починить deep link к данным**
     - Перевести внутренние вкладки настроек на URL‑состояние:
       - вариант A: `/account/settings?tab=data`
       - вариант B: `/account/settings/data` (дочерние роуты)
     - Тогда “server pill” сможет вести на корректный URL без зависимости от DOM‑якоря.
   - **Укрепить Lists hub**
     - Уйти от `url.includes(...)` к route data / дочерним роутам для статусов/коллекций.
   - **Тест‑контракт**
     - Добавить e2e: таблица редиректов + smoke по вкладкам аккаунта.

6. **Как должно быть (идеальный UX)**
   - В шапке минимум пунктов, но они **правильно названы**:
     - `Главная` = “найти фильм/сделать действие”
     - `Моё`/`Аккаунт` = “мои разделы”
   - Внутри хаба — 4–5 вкладок максимум, каждая с **1 строкой purpose** вверху:
     - “Сегодня: выбери что смотреть за 2 минуты”
     - “Дневник: запиши просмотр”
     - “Списки: сохрани на потом/собери коллекции”
     - “Входящие: релизы и уведомления”
     - “Настройки: подключение, приватность, предпочтения”
   - Любой статус‑элемент в UI (“Server pill”) ведёт прямо к месту, где это чинится, без сюрпризов.

7. **Важные технические детали**
   - Редиректы `redirectTo` могут не сохранять query/fragment — если важно, перейти на redirect через `UrlTree` с переносом параметров.
   - Якоря внутри вкладок ненадёжны, если контент вкладки не в DOM. Для deeplink нужен URL‑driven tab.
   - Для a11y:
     - активные ссылки должны иметь `aria-current="page"`,
     - модалки/bottom‑sheets обязаны иметь focus-trap + Escape + restore focus,
     - “tablist/tab/tabpanel” нельзя использовать без полного WAI‑ARIA паттерна.
